# DramaFlow 目前已实现

> 最后更新：2026-04-05

## 已打通的主链路

### 认证与协作

- JWT 登录、刷新、登出、忘记密码、重置密码、个人资料更新已可用。
- 团队 CRUD、团队成员管理、团队邀请链接已可用。
- 项目 CRUD、项目成员邀请、项目邀请接受、待处理邀请列表已可用。

### 文档、审核与评论

- Script、Storyboard、World Bible、Image、Video、Audio 文档模型已统一进入版本体系。
- 版本创建、恢复、提审、审批、驳回已闭环。
- 审核策略已经接入按内容类型的 `resolveContentReviewRequired()` 与 `canAutoApprove()`。
- 审核意见会写入审计记录；评论系统已支持 `parentId` 线程回复。

### AI 生成与媒体生产

- Synopsis、Script、Storyboard、Rewrite 已支持 SSE 流式生成。
- 图片、视频支持单镜头生成与批量任务创建。
- 单镜头媒体支持多候选版本，并可显式采纳指定版本。
- Prompt 预览链路已接入世界观角色、地点、风格注入。

### TTS

- TTS 任务会读取 World Bible 中保存的角色音色配置，而不是硬编码默认音色。
- 已提供角色音色配置 UI、音色样例播放、单镜头 TTS、按场景批量 TTS。
- 批量 TTS 完成后会为对应镜头生成 Audio 文档与版本。

### 时间线、导出与通知

- 时间线支持读取、保存、自动组装。
- 导出任务支持提交、进度更新、导出记录查询。
- FFmpeg 可用时会执行真实导出；显式允许时可回退为 mock 导出。
- 通知中心、未读数、标记已读/全部已读已可用。
- WebSocket 已推送 `job.updated`、`review.updated`、`notification.created`，轮询继续保留为降级路径。

## 当前运行前提

- 运行时仍然使用 `DevDatabaseService` 的 JSON 存储。
- Worker 仍然通过内部 API 轮询领取任务，并未迁移到 BullMQ。
- Prisma schema 已对齐到当前共享契约的大部分核心模型，但仍是目标生产模型，不是 live path。

## 当前接口形态

- `GET /projects/:id`：只返回工作区 summary。
- `GET /projects/:id/versions`：按需拉取版本数据。
- `GET /projects/:id/jobs`：按需拉取任务列表。
- `GET /projects/:id/timeline`：按需拉取时间线。
- `GET /projects/:id/exports`：按需拉取导出记录。
- `POST /project-invites/:id/accept`：接受项目邀请。
- `POST /scenes/:id/batch-tts-jobs`：按场景创建批量 TTS 任务。

## 仍处于开发态、尚未生产化的部分

- Prisma 还没有接入真实运行时读写。
- Redis / BullMQ 还没有替代当前轮询 Worker。
- Provider 生态仍以现有 OpenAI 兼容链路、Gemini 图片链路和 TTS 适配为主，更多第三方适配器尚未扩展。
- Web 端与 Worker 端还没有独立测试脚本；当前测试重点仍是 API 与 shared。

## 下一阶段更适合继续推进的方向

1. 将队列抽象从“轮询 Worker”平滑迁移到 BullMQ，同时保持现有 API 边界不变。
2. 把 Prisma 接到 live path，并补迁移脚本与真实数据库回归验证。
3. 继续补强可观测性，例如任务耗时、provider 调用日志、导出耗时和失败原因聚合。
4. 针对时间线编辑器补拖拽、裁剪、轨道级编辑与更完整的预览能力。
