# DramaFlow 目前已实现

> 最后更新：2026-05-09

## 已打通的主链路

### 认证与协作

- JWT 登录、刷新、登出、忘记密码、重置密码、个人资料更新已可用。
- 登录支持 IP 维度速率限制（15 分钟内最多 10 次尝试）。
- 团队 CRUD、团队成员管理（添加/移除/角色变更）、团队邀请链接（创建/列表/吊销/查询/接受）已可用。
- 项目 CRUD、项目成员邀请、项目邀请接受、待处理邀请列表已可用。
- 平台管理员仪表盘、团队管理员仪表盘已可用。

### 剧本编辑器

- 纯文本剧本编辑器：场景 CRUD、对白管理、角色列表内联编辑（添加/移除/修改角色名和简介）。
- 富文本剧本编辑器：基于 TipTap，支持场景标题（SceneHeading）、对白块（DialogueBlock）、导演备注（DirectorNote）自定义扩展，结构化剧本编辑。
- 角色列表内联编辑，可直接在编辑器中增删角色。

### 分镜工作台

- 分镜编辑器已支持完整的镜头数据展示与编辑。
- 分镜工作台：按场景分组的镜头卡片列表，镜头详情侧边抽屉，场景级别批量 TTS。
- 批量生成：支持对缺失图片的镜头进行批量图片生成，工具栏集成批量生成控制。
- 景别和运镜方式的中英文别名映射与本地化标签已完整。

### 文档、版本、审核与评论

- Script、Storyboard、World Bible、Image、Video、Audio、Subtitle 文档模型已统一进入版本体系。
- 版本创建、恢复、提审、审批、驳回已闭环。
- 草稿版本支持编辑内容与删除。
- 独立版本管理面板：左右分栏布局，左侧版本列表（支持状态过滤），右侧内容预览与版本对比。
- 审核策略已接入按内容类型的 `resolveContentReviewRequired()` 与 `canAutoApprove()`。
- 审核配置面板：按内容类型独立配置是否需要审核、可自动通过的角色列表。
- 审核意见会写入审计记录；评论系统已支持 `parentId` 线程回复。

### AI 生成与媒体生产

- Synopsis、Script、Storyboard、Rewrite 已支持 SSE 流式生成。
- AI 生成面板支持"先生成大纲"流程，SSE 流式输出。
- 图片、视频支持单镜头生成与批量任务创建。
- 单镜头媒体支持多候选版本，并可显式采纳指定版本。
- Prompt 预览链路已接入世界观角色、地点、风格注入。
- Provider 选择器：生成时可选择使用个人或团队的图片/视频 Provider。

### 多 Provider 支持

- 已接入 7 个 AI Provider 实现：
  - **文本生成**：OpenAI 兼容 completions（支持流式）。
  - **图片生成**：OpenAI 兼容、Google Gemini（文生图 + 参考图编辑）、Stable Diffusion WebUI、ComfyUI（工作流模式）、Grok。
  - **视频生成**：OpenAI 兼容、Grok。
  - **TTS**：语音合成（含 Mock 适配器）。
- Provider 配置表单：通用的 Provider 添加/管理表单，支持配置各 Provider 专用参数（SD 采样器/步数、ComfyUI 工作流 JSON、Grok 宽高比/时长等）。
- 用户和团队均可独立管理多套图片/视频 Provider 配置，并指定默认 Provider。

### 媒体绑定与字幕

- 分镜内容中的 `mediaBindings` 支持每个镜头独立绑定图片版本、视频版本、音频版本和字幕。
- 媒体与字幕已集成到分镜版本管理流程中。
- 草稿版本的媒体绑定可独立更新。

### TTS

- TTS 任务会读取 World Bible 中保存的角色音色配置，而不是硬编码默认音色。
- 已提供角色音色配置 UI、音色样例播放、单镜头 TTS、按场景批量 TTS。
- 批量 TTS 完成后会为对应镜头生成 Audio 文档与版本。

### 时间线编辑器

- 可视化时间线编辑器：视频、对白、音乐、音效、字幕五类轨道。
- 缩放控制（20-200 像素/秒），播放/暂停控制。
- 从媒体库拖拽素材到时间线轨道。
- 自动组装：根据分镜数据自动填充时间线。
- 保存时间线。
- 导出为 MP4/MOV/WebM，FFmpeg 可用时执行真实导出。

### 媒体库

- 侧边媒体库面板：视频、音频、字幕、图片四个标签页。
- 资产卡片展示、预览、上传功能。
- 字幕文本预览。
- 支持注册项目资产。

### 世界观编辑器

- 角色档案管理（外观、性格、标签、参考图、服装）。
- 场景地点管理（描述、光照、时间段、参考图）。
- 视觉风格指南（风格描述、色彩方案、构图备注、负面提示词、参考图）。
- 角色音色配置（TTS Provider、语音 ID、语速/情感/音量参数）。
- AI 参考图生成对话框：支持为角色、地点、风格指南生成 AI 参考图。

### 时间线、导出与通知

- 时间线支持读取、保存、自动组装。
- 导出任务支持提交、进度更新、导出记录查询。
- FFmpeg 可用时会执行真实导出；显式允许时可回退为 mock 导出。
- 通知中心、未读数、标记已读/全部已读已可用。
- WebSocket 已推送 `job.updated`、`review.updated`、`notification.created`，轮询继续保留为降级路径。

### 国际化

- 工作区面板和 Provider 表单支持中文/英文切换。
- 景别和运镜方式已完整支持中文/英文本地化标签。

## 当前运行前提

- 运行时仍然使用 `DevDatabaseService` 的 JSON 存储。
- Worker 仍然通过内部 API 轮询领取任务（可配置轮询间隔），并未迁移到 BullMQ。
- Prisma schema 已对齐到当前共享契约的大部分核心模型，但仍是目标生产模型，不是 live path。

## 当前接口形态

- `POST /auth/register` — 用户注册。
- `POST /auth/login` — 登录（含速率限制）。
- `POST /auth/refresh` — 刷新令牌。
- `POST /auth/logout` — 登出。
- `POST /auth/forgot-password` — 发起密码重置。
- `POST /auth/reset-password` — 执行密码重置。
- `GET /auth/me` — 获取当前用户资料。
- `PATCH /auth/me` — 更新个人资料（含 LLM 配置、Provider 配置、默认 Provider）。
- `POST /auth/me/llm-models` — 列出可用 LLM 模型。
- `GET /projects/:id` — 返回工作区 summary。
- `GET /projects/:id/versions` — 按需拉取版本数据。
- `GET /projects/:id/jobs` — 按需拉取任务列表（支持状态/类型/batchId 过滤）。
- `GET /projects/:id/timeline` — 按需拉取时间线。
- `PUT /projects/:id/timeline` — 保存时间线。
- `POST /projects/:id/timeline/auto-assemble` — 自动组装时间线。
- `GET /projects/:id/exports` — 按需拉取导出记录。
- `POST /projects/:id/assets` — 注册项目资产。
- `POST /project-invites/:id/accept` — 接受项目邀请。
- `POST /scenes/:id/batch-tts-jobs` — 按场景创建批量 TTS 任务。
- `POST /projects/:id/jobs/batch-image` — 批量图片生成任务。
- `POST /projects/:id/jobs/batch-video` — 批量视频生成任务。
- `GET /jobs/:id` — 获取任务状态。
- `POST /jobs/:id/cancel` — 取消任务。
- `POST /jobs/:id/retry` — 重试失败任务。
- `GET /jobs/voices` — 列出可用 TTS 音色。
- `POST /jobs/export` — 创建导出任务。
- `POST /uploads` — 创建上传目标（直传流程）。
- `PUT /uploads/direct/:key` — 直接上传文件。
- `GET /assets/:id/url` — 获取资产访问 URL。
- `GET /admin/platform/overview` — 平台级概览指标。
- `GET /admin/teams/:id/overview` — 团队管理仪表盘。
- `GET /admin/teams/:id/settings` — 团队设置（LLM、图片生成配置）。
- WebSocket 网关：`project.subscribe` / `project.unsubscribe` 加入/离开项目房间。

## 仍处于开发态、尚未生产化的部分

- Prisma 还没有接入真实运行时读写。
- Redis / BullMQ 还没有替代当前轮询 Worker。
- Web 端与 Worker 端还没有独立测试脚本；当前测试重点仍是 API 与 shared。
- 测试覆盖：shared 包业务规则测试、数据库初始化测试、Provider 测试（文本/媒体/Gemini 共 9 个用例），整体测试覆盖仍较薄。

## 下一阶段更适合继续推进的方向

1. 将队列抽象从"轮询 Worker"平滑迁移到 BullMQ，同时保持现有 API 边界不变。
2. 把 Prisma 接到 live path，并补迁移脚本与真实数据库回归验证。
3. 继续补强可观测性，例如任务耗时、provider 调用日志、导出耗时和失败原因聚合。
4. 针对时间线编辑器补拖拽裁剪、轨道级精细编辑与更完整的预览能力。
5. 扩充测试覆盖，特别是 Web 端组件测试和 Worker 端 Provider 集成测试。
